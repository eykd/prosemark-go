# Project State Summary ‚Äî 2026-03-03

## Executive Summary

`prosemark-go` is a Go CLI tool (`pmk`) for managing long-form prose projects using a binder-based outline strategy. As of this date, two major features have been fully implemented and merged to `master` ‚Äî **001-prosemark-binder** (core parse/add/delete/move operations) and **002-node-identity** (project init, node creation, editor integration, integrity validation) ‚Äî followed by a bug-fix sprint **003-smoke-test-bugs** that addressed CLI output correctness, error propagation, and spurious diagnostic suppression. The codebase passes all quality gates with 97.1% overall test coverage (100% of non-Impl functions covered) and hosts 31 source files across 7 packages, backed by 73 test files.

## CLI Interface

All subcommands accept `--project <dir>` to specify the project directory; defaults to `$CWD`. The binder file is always `_binder.md` within that directory.

| Command | Flags | Output |
|---------|-------|--------|
| `pmk parse` | `--project <dir>` | JSON to stdout |
| `pmk add-child` | `--project`, `--parent`, `--target`, `--title`, `--first/--at/--before/--after`, `--force`, `--new` | Confirmation to stdout; diagnostics to stderr |
| `pmk delete` | `--project`, `--selector`, `--yes` | Confirmation to stdout; diagnostics to stderr |
| `pmk move` | `--project`, `--source`, `--dest`, `--yes`, `--first/--at/--before/--after` | Confirmation to stdout; diagnostics to stderr |
| `pmk init` | `--project` | Initializes `_binder.md` and `.prosemark.yml` |
| `pmk edit` | `--project`, `--selector` | Opens node file in `$EDITOR` |
| `pmk doctor` | `--project` | Audit diagnostics to stdout/stderr |

**Output convention**: Human-readable confirmations go to stdout; diagnostic messages (errors and warnings with codes) go to stderr. Exit 1 on any error diagnostic.

**`--json` flag removed**: As of the 003-smoke-test-bugs sprint, all commands emit human-readable output by default. The conformance runner uses `binder.OpResult` (structured JSON type) rather than a `--json` CLI flag.

## Domain Model

### Core Types

| Type | Package | Key Fields |
|------|---------|-----------|
| `Node` | `internal/binder` | `Type` ("root"\|"node"), `Target` (resolved path), `Title`, `Children` (ordered, never nil), source metadata: `Line`, `Column`, `ByteOffset`, `EndLine`, `SubtreeEnd`, `Indent`, `IndentChar`, `ListMarker`, `RawLine`, `InCodeFence` |
| `Diagnostic` | `internal/binder` | `Severity` ("error"\|"warning"), `Code` (e.g. "BNDE001"), `Message`, `Location` (line/col/byte, optional) |
| `ParseResult` | `internal/binder` | `Version` ("1"), `Root` (*Node), `Lines` (original source), `LineEnds`, `RefDefs`, `HasBOM`, `HasPragma`, `PragmaLine` |
| `OpResult` | `internal/binder` | `Version` ("1"), `Changed` (bool), `Diagnostics` ([]Diagnostic) |
| `NodeId` | `internal/node` | string alias (UUID v7) |
| `Frontmatter` | `internal/node` | `ID` (UUID v7), `Title`, `Synopsis` (optional strings), `Created`, `Updated` (RFC3339) |
| `AuditCode` | `internal/node` | AUD001‚ÄìAUD008, AUDW001, BNDW001 |

### Selector Syntax

Selectors navigate the binder tree:
- Bare stem: `chapter` matches `chapter.md`
- Index qualifier: `chapter[1]` picks the second match
- Chain: `part1:chapter` navigates to child of `part1`
- Root: `.` refers to the binder root

### Diagnostic Codes

| Category | Codes | Severity | Trigger |
|----------|-------|----------|---------|
| Parse errors | `BNDE001`‚Äì`BNDE003` | error (exit 1) | Illegal path chars, path escapes root, ambiguous wikilink |
| Parse warnings | `BNDW001`‚Äì`BNDW010` | warning (exit 0) | Missing pragma, various lint warnings |
| Operation errors | `OPE001`‚Äì`OPE010` | error (exit 1) | No match, ambiguous selector, cycle, invalid target, target is binder, node in fence, sibling not found, index OOB, I/O failure, conflicting flags |
| Operation warnings | `OPW001`‚Äì`OPW005` | warning (exit 0) | Multi-match, duplicate skipped, non-structural content destroyed, empty sublist pruned, cascade delete |
| Audit errors | `AUD001`‚Äì`AUD008` | error | Missing file, orphaned node, duplicate ref, ID mismatch, missing frontmatter, empty file, unparseable YAML, missing/invalid config |
| Audit warnings | `AUDW001`, `BNDW001` | warning | Non-UUID filename, binder missing pragma |

Functions wrapping OS/exec calls are named with `*Impl` suffix and excluded from coverage calculation:

- `WriteTestFileImpl`, `WriteIRImpl`, `ReadIRImpl` ‚Äî acceptance pipeline I/O

These functions are exercised by integration and acceptance tests, not unit tests.

## ATDD Pipeline

```
specs/001-prosemark-binder/*.txt
specs/002-node-identity/*.txt
    ‚îÇ (just acceptance-parse)
    ‚ñº
acceptance-pipeline/ir/*.json        ‚Üê committed intermediate representation
    ‚îÇ (just acceptance-generate)
    ‚ñº
generated-acceptance-tests/*_test.go ‚Üê committed generated test stubs
    ‚îÇ (just acceptance-run)
    ‚ñº
Test results (Go test output)
```

**Stub preservation**: The generator (`acceptance/generator.go`) detects bound (implemented) test functions and preserves their bodies across regeneration ‚Äî only unbound stubs (containing `t.Skip("acceptance test not yet bound")`) are overwritten.

**Pipeline CLI**: `go run ./acceptance/cmd/pipeline -action=<parse|generate|run>`
